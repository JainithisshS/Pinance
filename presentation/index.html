1 Problem Statement

Users Struggle With

  • Tracking actual spending from bank SMS / UPI notifications
  • Understanding personal financial risk (EMI burden, savings, fixed vs variable)
  • Interpreting financial news in context of their own profile
  • Getting personalized, actionable financial guidance

Existing Apps Are Siloed

📱

Expense Trackers

Only track spending — no insights or news

📰

Market Apps

Generic news — not personalized to user

💡

Advisory Apps

Cookie-cutter advice — no real data integration

Gap: No system combines personal data + news sentiment + domain knowledge into explainable, agentic recommendations

2 Proposed Solution

End-to-end Multi-Agent AI Platform with a Two-Tier Pipeline: Traditional ML (Tier 1) + LLM Agentic Reasoning (Tier 2)

🧠 Agent A

Finance Analyzer

Random Forest risk model + Groq LLM personalized insights. Splits income/expenses, computes savings rate, projects monthly risk.

📊 Agent B

News Analyzer

FinBERT NLP sentiment + spaCy NER + trend forecasting. Live RSS news feed from Economic Times + NSE market data.

💬 Agent C

Decision Chatbot

RAG retrieval (32 knowledge chunks, TF-IDF) + Groq LLM multi-turn chatbot. Combines Agent A + B context.

📚 Learning

Adaptive Micro-Learning

Bayesian knowledge tracing + AI-generated cards + DAG concept graph (10 concepts, 4 levels).

📱

Android App

Kotlin + Jetpack Compose. Notification listener auto-captures bank SMS. Dashboard, Chat, Learning screens.

☁️

Cloud Deployment

Docker + Render.com + Supabase (PostgreSQL) + Firebase Auth. Dual-DB with auto-fallback.

Amount Extraction (3-Tier Regex)

Tier 1: Rs/INR/₹ + number "Rs 500", "INR 1,250.50", "₹750" Tier 2: Number + "rupees" "500 rupees" Tier 3: After keyword + number "debited 1000", "Amount: 500" Fallback: debit/credit + up to 15 chars + number Last resort: any 2+ digit standalone number

Spam Filtering

  • Reject: OTPs, promos, "cashback offer", "subscribe now"
  • Require: money amount + (debit/credit keyword OR bank signal)
  • Bank signals: "a/c", "upi", "neft", "imps", "xxxx"

6 Agent A — Personal Finance Analyzer

🧠 Agent A · Tier 1 ML + Tier 2 LLM

Tier 1: Random Forest Risk Model

PropertyValue
AlgorithmRandomForest (120 trees, depth 6)
Training900 synthetic monthly profiles
Featuresincome, fixed_exp, var_exp, savings, savings_rate, variable_share
Outputlow / medium / high + confidence %

Key Innovation: Month Proration

  • Projects partial-month data → full-month equivalent
  • Prevents false "low risk" early in month
  • Confidence downgraded: <30% coverage → medium max, <15% → low forced

Tier 2: Groq LLM Insight

System Prompt

"You are Agent A, a personal finance analyst AI for an Indian user. Produce a short, personalized, actionable financial insight. Be specific about numbers. Use INR (₹). 4-5 sentences."

Additional Features

  • Fixed vs Variable expense classification by category
  • Human-readable risk explanations (savings rate + variable share)
  • Risk evaluations logged to risk_logs table for model tuning
  • Separate expense_breakdown endpoint for dashboard charts

FinBERT NLP Pipeline

  • Model: ProsusAI/finbert (BERT fine-tuned for finance)
  • Scoring: Headline (40%) + Body (60%) weighted
  • Output: P(positive), P(negative), P(neutral)
  • Scale: 10 × (P(pos) − P(neg)) → [-10, +10]
  • Fallback: 27 positive + 31 negative keywords + strong phrases

Entity & Sector Detection

  • spaCy NER → ORG entities (with noise filtering)
  • 25+ company-to-sector mappings (Banking, IT, Auto, FMCG...)
  • Percentage & points move extraction via regex

8 Agent C — RAG Decision Chatbot

💬 Agent C · RAG + Groq LLM Multi-Turn

Two-Tier Pipeline

Tier 1: RAG Retrieval

Query built from Agent A + Agent B + user question → TF-IDF score against 32 knowledge chunks → top-3 retrieved

↓

Tier 2: Groq LLM Chatbot

System + context injection + RAG knowledge + up to 10 conversation turns → personalized recommendation

↓ if LLM fails

Deterministic Fallback

Keyword matching → topic-specific advice (budget, invest, debt, emergency, risk, what-if)

9 Adaptive Micro-Learning System

📚 Bayesian Knowledge Tracing + AI Card Generation

Knowledge Graph (DAG)

Level 1: money_basics income_basics │ │ Level 2: expense_tracking → budgeting_basics ┌─────┼──────┐ Level 3: emergency debt saving _fund _mgmt _strategies │ │ │ Level 4: investment credit financial _basics _score _goals

Bayesian Update Rules

ScenarioUnknownPartialMastered
Fast correct (<30s)−0.4+0.1+0.3
Slow correct−0.3+0.15+0.15
Fast incorrect+0.2+0.1−0.3
Slow incorrect (>60s)+0.4−0.2−0.2

Curriculum Compiler

score = readiness × urgency × relevance

readiness = min(mastery of prerequisites)
urgency = 1 − P(Mastered)
relevance = context multiplier: high risk → budget ×1.5, has debt → debt ×1.6

AI Card Generation (Groq)

  • Content: 150-200 word micro-lesson + practical examples
  • Quiz: 4-option MCQ with explanation (JSON structured output)
  • Cards cached by concept; invalidated on mastery level change
  • Background pre-generation for instant delivery

16 Deployment & DevOps

Docker Container

FROM python:3.11-slim # PyTorch CPU + spaCy + transformers EXPOSE 7860 CMD uvicorn backend.main:app --host 0.0.0.0 --port 7860

Render.com (render.yaml)

  • type: web, runtime: docker, plan: free
  • Environment: SUPABASE_URL, SUPABASE_SERVICE_KEY
  • GROQ_API_KEY, FIREBASE_ADMIN_SDK_JSON

Startup Optimizations

  • FinBERT preloaded in background thread
  • NSE market data pre-cached at startup
  • Learning cards pre-generated asynchronously
  • ML models lazy-loaded on first request

Graceful Degradation

  • No Groq key → Tier 1 ML still works
  • No Supabase → SQLite fallback
  • No Firebase → default_user dev mode
  • No FinBERT → keyword sentiment fallback
  • No spaCy → dictionary-based NER
